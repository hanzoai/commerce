The Crowdstart API is designed to handle all of the most common tasks with associated with running a crowdfunding campaign.

The API is currently in beta. Please contact support@crowdstart.com if you intend to use our API in any production code.

Data types

  • Currency is always in cents.
  • Dates are always RFC3339.

Object reference

This section contains specifications for the public-facing objects that are used by the API.

Campaign

This represents a single fundraising campaign.

Field name Field Description Type
organizationId The unique identifier for this campaign string
approved A switch indicating whether or not the campaign is approved for launch. Useful for 'soft launches' and build-up to launch day. bool
enabled A switch indicating whether or not this campaign is currently enabled for use in the system. Campaign will not appear if it is not enabled. bool
category A representation of the general style of campaign (Technology, Arts, Entertainment, etc). This is a string that relates the campaign type. Category
title The title of the campaign. string
tagline The motto or tag line for the campaign. string
pitchMedia A representation (usually a link) to media that is to be used in the campaign's pitch. string
videoUrl A url to the video associated with the campaign. string
videoOverlayUrl A url to the overlay to be placed over the video before it begins - this is typically a company logo or a picture of the product. string
imageUrl A banner or logo for the project itself. string
description The long-form explanation of the campaign. string
backers The number of backers that have contributed to the campaign. int
raised The amount of money raised in the campaign. int64
thumbnail A smaller resource url to represent the campaign in a list. string
originalUrl An URL pointing to the previous phase of the campaign (such as an Indiegogo or Kickstarter campaign) if applicable. string
storeUrl A URL leading to the store associated with the campaign, if applicable. string
productIds A list of ids of products/perks that are offered by the campaign. []string
googleAnalytics The Google Analytics tag for the campaign. string
facebookTag The social network tag for the campaign. string
links Important links related to the campaign not covered in other areas (such as media releases). []string

Campaign-specific objects

These are objects that are used in the Campaign entity, but are not shared by other objects.

Category

This is string that expresses the general classification or division your campaign falls under.

This list is under constant re-evaluation, but assigned values will never change. If you would like to request a change to this list, contact us at support@crowdstart.com.

Value Description
"arts" Arts
"fashion" Fashion
"film" Film
"food" Food
"gaming" Gaming
"health" Health
"music" Music
"sports" Sports
"technology" Technology

LineItem

This object is a representation of one particular instance of a customer requesting a Product. It contains some fields associated with a Product variant. This can be either a prospective request (in the case of Cart), or a confirmed one (in the case of Order)

Field Name Field Description Type
productId The specific identifier for the ProductId referenced in this LineItem. This is a computed field. string
productName The specific identifier for the ProductName referenced in this LineItem. string
productSlug The specific identifier for the Product referenced in this LineItem. This is a computed field, related to the data from the Product property (see below) string
collectionId The collection ID. string
variantId The specific item type ID or perk type being requested. string
variantName The specific item type Name or perk type being requested. string
variantSKU The specific item type variant SKU or perk type being requested. string
price The price of this LineItem in cents. int
quantity The amount of the item/perk requested in this line. int
weight The item weight, typically in ounces. int
taxable Whether the product is subject to taxes. bool

Order

This object is a representation of a user's confirmed purchase. The order should not be confused with a Cart, which are prospective purchases.

Field Name Field Description Type
createdAt The time the order first came to be. string
updatedAt The time the order was last modified (not necessarily by the user). string
id The internal unique identifier for this order. string
userId The identifier for the user who placed this order. string
shipping Charges for mailing the order in cents int64
tax Charges on the order levied by any relevant governments in cents int64
subtotal Charges from actual purchases in cents int64
total Net charge to be sent to the client cents. Includes Subtotal, Tax, and Shipping. int64
items A list of LineItems that comprise this order. []LineItem
campaignId The unique identifier for the fundraising campaign that spawned this order (if applicable) string
preorder A switch indicating whether or not this is a preorder. bool
cancelled A switch indicating whether or not this order has been cancelled. bool
shipped A switch indicating whether or not this order has been shipped. bool
test A switch indicating whether or not this is a test order. bool

Product-specific objects

These are objects that are used in the Product entity, but are not shared by other objects.

The Media object represents a graphic resource to be used with advertising a Product.

Field Name Field Description Type
type The type of the media MediaType
alt The alt-text for the media. string
url The URL where the media is hosted. string
x The width of the media, in pixels. int
y The height of the media, in pixels. int

  "url": "http://suchtees.com/media/top/2.jpg",
  "x":   400,
  "y":   100

This is an enum which describes the type of a media resource.

Value Description

Option

A key-value pair of metadata about a Product variant

Field Description Type
name The name of the datum string

Variant

The Variant object signifies specifics about an individual kind of Product.

Field Name Field Description Type
id The unique internal identifier for this variant. string
sku The unique external product identifier for this variant (helps with fulfillment) string
name Display name for this variant string
price The price of this variant in cents int64
inventory How many of the variant are currently in stock. int
weight How much the variant weighs. This is typically in ounces, but can be in grams, miligrams, or whatever else as long as the usage is consistent. float
dimensions The height by width by depth of the variant, physically. Typically in inches, but handles metric measurements fine with consistent use. string
options Metadata associated with the variant. []Option

The options object may vary.

This object represents someone who is registered with the Crowdstart platform. This is usually NOT a backer or buyer - this is typically a campaign owner, manager, etc.

CouponType

CouponType is a string which represents how the discount should be applied.

Value Description
flat Discounts amount in cents from the subtotal
percent Discounts amount as a percent of subtotal
free_shipping Discounts the cost of shipping from the total

Authentication

The access token required to use any of the below API requests can be found on the Crowdstart dashboard. All endpoints are secured with and require SSL/HTTPS. To authenticate, include the access token the URL, at the end of the API path. This takes the form of:

GET /api/campaign2/<access token>
GET /api/coupon2/<access token>

Or whichever path is appropriate for the entity you are working with. Access tokens can be retrieved from your user dashboard. The 'live' tokens are the tokens for use in your production environment while 'test' tokens function in your sandbox.

Other reminders: - All requests must be made over HTTPS. - All requests expect and return JSON. Please set Content-Type and Accept to application-json. - All money measurements, per the object definitions, are in cents, and are returned only in whole numbers (no decimal places). Therefore, $1 USD will be returned as 100, and $.01 USD will be returned as 1.

API Errors

Not all requests will be successful. In the event of any problems, an error object will be returned from Crowdstart with the best message we can give. Other optional information, such as codes or error types, will be provided if possible.

"error": {
  "message": "No access token provided.",
  "type": "api-error"
}
"error": {
  "message": "Access token is not valid."
}
"error": {
  "message": "Service not found."
  "type": "service-error"
  "code": "404"
}

Pagination

Many calls can potentially return hundreds of objects. To avoid sending massive amounts of data over the wire, the API uses the following format on its pagination object:

Field Name Field Description Type
total The number of records available. int
limit The maximum number of records power page. Default is 100. int
offset The index of the first record returned. int
page The current page number. int
pages The number of pages available. int

Example of a response with pagination available:

{
  "paging": {
    "total":    1532,
    "limit":    150,
    "offset":   300,
    "page":     3,
    "pages":    11,
  },
  "orders": []
}
